# GrimbaNews — OSS Community Recognition Program Plan

**Status:** plan v0
**Owner:** Maria Lopez (Community) + Michael O'Connor (Tech Writer)
**Walks:** Mythos S2087 (community — recognition program: CONTRIBUTORS.md + monthly shout-outs) deferred → partial
**Gating dependency:** S2043 public OSS org provisioning + at least one external contributor.

## Why this exists

OSS contributors are scarce and easily lost. A predictable recognition cadence (monthly shout-outs, CONTRIBUTORS.md, contributor badges) is the cheapest retention lever.

## v1 program

| Recognition | Format |
|---|---|
| First contribution | Welcome PR comment + name added to CONTRIBUTORS.md |
| 5+ merged PRs | Contributor tier on GitHub + small social shout-out from @grimbanews |
| 20+ merged PRs / sustained | Maintainer-level access + named credit in next annual transparency report |
| Major feature contribution | Featured contributor section in monthly newsletter |
| Long-tenure contributor (1yr+) | Optional listing in `/equipe/contributeurs-open-source` page |

## Cadence

- Monthly: Maria assembles shout-out from PRs/issues merged that month.
- Quarterly: CONTRIBUTORS.md cleanup + new-tier promotions.
- Annual: top-contributor section in transparency report.

## Anti-patterns

- No leaderboards (toxic gamification).
- No "contributor of the month" competitions (zero-sum hostility).
- No public ranking of contributors by PR count.
