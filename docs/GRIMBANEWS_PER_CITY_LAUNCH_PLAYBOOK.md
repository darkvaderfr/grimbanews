# GrimbaNews — Per-City Launch Playbook

**Status:** plan v0
**Owner:** Lucy Leai (Strategy) + per-region editor + per-city contact
**Walks:** Mythos S1610 (per-city launch playbook) deferred → partial
**Gating dependency:** S1601 (cities table) + S1608 (landing) + S1609 (per-city ads).

## Phase 0 — operator prep

1. City selected (by per-region editor based on source density + reader demand).
2. City row inserted in `local_cities`.
3. Per-city editor brief drafted + Lucy review.
4. Per-city source roster confirmed (≥ 3 sources).
5. Per-city ad inventory established (if paid).

## Phase 1 — soft launch (30 days)

1. `/local/{city-slug}` page live but unannounced.
2. Per-region editor monitors per-city cluster + bias mix daily.
3. Per-city reader-feedback collected via widget (per Wave AAMM).
4. Per-city engagement metrics: reads, shares, return rate.

## Phase 2 — external announcement

1. Per-city press-release to local press.
2. Per-city social-media post per Wave AALL platform syndication.
3. Per-city ad inventory opened (if applicable).

## Phase 3 — quarterly review

1. Per-city quarterly editorial retro.
2. Per-city ad-revenue review (if applicable).
3. Per-city expansion / consolidation decision.

## Failure conditions

- < 50 unique readers/week after 30 days → city under-served / wrong city.
- Bias-mix imbalance > 70% after 60 days → source-roster gap.
- Per-city ad-revenue < hosting cost after 12 months → consolidate.
